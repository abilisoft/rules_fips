# Security policy

## Supported code

`rules_fips` is pre-release software. Security fixes are made on the default
branch; no released maintenance branches are currently supported.

## Report privately

Do not open a public issue for a suspected vulnerability. Use
[GitHub private vulnerability reporting](https://github.com/abilisoft/rules_fips/security/advisories/new)
or email `oss@abilisoft.com` with:

- the affected target and architecture;
- a minimal reproduction or evidence;
- expected and observed behavior; and
- any known disclosure deadline.

Please omit credentials, production data, and regulated material. You should
receive an acknowledgement within five business days. We will coordinate
validation, remediation, and disclosure based on severity and impact.

## Scope note

A disagreement about CMVP applicability or an unlisted operational environment
is important, but it is not automatically a software vulnerability. Report
incorrect enforcement, bypasses, undeclared runtime dependencies, integrity-pin
failures, unsafe extraction, or misleading generated evidence privately.

This policy does not represent a FIPS certification or compliance guarantee.
