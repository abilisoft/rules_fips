# Support

Use [GitHub Issues](https://github.com/abilisoft/rules_fips/issues) for
reproducible build failures, documentation problems, and feature proposals.
Include the Bazel version, target label, platform config, failing
mnemonic, and a redacted error log.

Security reports belong in [private vulnerability reporting](SECURITY.md), not
in a public issue.

Maintainers can explain what the project builds and checks. They cannot decide
whether a particular organization or deployment satisfies regulatory
requirements.
